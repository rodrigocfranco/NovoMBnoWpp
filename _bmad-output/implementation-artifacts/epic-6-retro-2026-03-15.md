## 2. O Que Funcionou Bem

### Arquitetura Elegante — Feedback Fora do Pipeline LangGraph
Decisão de processar feedback diretamente no webhook handler, sem passar pelo grafo. Simples, rápido, sem complexidade desnecessária. Clique do aluno → save no banco → confirmação instantânea.

### Redis para Estado Temporário (Pending Comment)
Padrão `feedback_pending:{phone}` com TTL 5min — auto-expira, sem estados órfãos. Reutiliza o mesmo padrão do debouncing e deduplicação. Padrão validado e recomendado para estados conversacionais temporários.

### Buttons como Mensagem Separada
Body do interactive message limitado a 1024 chars — resposta do assistente frequentemente excede isso. Enviar buttons como mensagem separada permite formatação markdown rica na resposta + buttons limpos. Decisão pragmática que evitou problemas futuros.

### Code Review Adversarial Eficaz
9 findings (1 HIGH, 5 MEDIUM, 3 LOW) — todos corrigidos. O finding HIGH teria corrompido o fluxo de comentário em mensagens não-texto. Code review continua sendo o asset #1 de qualidade.

### Epic Enxuto = Qualidade Alta
1 story com foco permitiu entrega limpa. 67 testes novos para uma única story é cobertura sólida.

### Entregas Pendentes da Retro Anterior Concluídas

## 3. O Que Pode Melhorar

### 3.1 Dispatch de Tipos de Mensagem Continua Frágil
**Severidade: ALTA**

Finding H1: pending comment state perdido em mensagens não-texto (áudio/imagem consumiam a key Redis). Padrão recorrente — Epic 3 teve problemas similares no dispatch do `process_media`. O webhook handler precisa de hardening no roteamento por tipo de mensagem.

### 3.2 Retro Watch Items Regrediram
**Severidade: ALTA**

Story 6.1 NÃO tem a seção "Retro Watch Items" que foi acordada na retro Epics 3+5. Todas as stories do Sprint 6 (3.1, 3.2, 5.1, 5.2) tinham. Causa: story criada antes da retro, sem atualização retroativa. Mecanismo bom, mas faltou enforcement.

### 3.3 Code Review como Única Linha de Defesa
**Severidade: MÉDIA**

9 findings em 1 review. O code review adversarial é eficaz, mas continua sendo a ÚNICA camada que pega bugs de integração. Se o reviewer não perceber, o bug vai para produção.

### 3.4 "Integration Tests" Ainda com Mocks em Boundaries Externos
**Severidade: MÉDIA**

Finding M5: testes renomeados de "integração" para refletir que ainda mocam chamadas externas. Pendência que se arrasta desde Epic 1. Validação real com WhatsApp API continua sem evidência de execução.

### 3.5 Dados de Satisfação Quase Enviesados
**Severidade: MÉDIA**

Finding M4: "Comentar" defaultava para `rating="negative"`. Se não fosse pego no code review, os dados da North Star Metric estariam corrompidos. Corrigido para `rating="comment"` como categoria própria.
