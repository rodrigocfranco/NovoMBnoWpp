## 3. O Que Pode Melhorar

### 3.1 Over-Mocking (Risco Sistêmico)
**Severidade: CRÍTICA**

O code review da Story 1.3 revelou um bug que **passou em todos os testes**: cache key mismatch onde `get_session(phone)` e `cache_session(user_id)` usavam chaves diferentes. O cache NUNCA funcionou mas os testes passaram porque os mocks isolaram completamente a lógica.

**Ação:** Adicionar testes de integração leves que validem fluxos reais (sem mock de Redis/PostgreSQL) no Epic 2.

### 3.2 Error Handling Silencioso
**Severidade: ALTA**

Em 4 das 6 stories, encontramos padrões `except Exception` que engolem erros silenciosamente. Isso pode mascarar problemas em produção.

**Ação:** Revisar error handlers no início do Epic 2; logar exceções com structlog antes de fallback.

### 3.3 Debounce: Limitação Conhecida
**Severidade: MÉDIA**

Mensagens no buffer Redis são perdidas se a instância Cloud Run encerrar antes do timer disparar. Documentado como limitação aceitável para MVP.

**Ação:** Monitorar taxa de mensagens perdidas em produção; considerar Redis Streams se necessário.

### 3.4 Lint Issues Pendentes
**Severidade: BAIXA**

23 issues de Ruff (E501, F401, UP035) em 8 arquivos de teste e 2 arquivos de código.

**Ação:** Corrigir antes de iniciar Epic 2.
