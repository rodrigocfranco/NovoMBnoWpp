## 2. O Que Funcionou Bem

## 3. O Que Pode Melhorar

### 3.1 Camada de Cache Quebrou Testes Pré-Existentes
**Severidade: ALTA**

A adição do Redis cache (ADR-013) quebrou testes que não mockavam a camada de cache:
- `test_global_timeout` (bulas_med) — cache retornava resultado real, bypass do timeout
- `test_config_service_returns_error_fallback` — ConfigService agora usa Redis cache-aside

**Causa raiz:** Testes não isolavam dependência externa (Redis). Quando cache foi adicionado, testes que dependiam do fluxo sem cache quebraram silenciosamente.

**Correção aplicada na retro:** Mock do `cache_service` e `get_redis_client` nos testes afetados. Ambos passando.

### 3.2 Over-Mocking Continua Sendo Risco
**Severidade: MÉDIA**

Finding H2 (Story 7.1): todos os testes de `tracked_tools` usavam mocks completos, escondendo bugs de integração entre wrapper e ToolNode real. 6 de 7 findings HIGH nas retros anteriores foram causados por over-mocking.

### 3.3 Dispatch de Tipos de Mensagem — Pendência Carregada
**Severidade: MÉDIA**

Action item herdado da retro Epic 6 (e Epics 3+5 antes disso). O webhook handler continua com lógica de roteamento por tipo de mensagem que é superfície recorrente de bugs. Não endereçado no Epic 7 (fora de escopo).

### 3.4 Erros de Lint Pré-Existentes
**Severidade: BAIXA**

4 erros de ruff pré-existentes em arquivos não modificados (E501 line too long em `system.py`, `bulas_med.py`; UP041 em `orchestrate_llm_streaming.py`). Não bloqueantes, mas acumulam dívida técnica.

## 4. Key Insights

1. **Observabilidade muda a maturidade do projeto** — Com CostLog, Langfuse e MetricsService, mb-wpp tem infraestrutura profissional de debugging e monitoramento

5. **Validação real com APIs foi feita** — ADR-013 prova que o pipeline funciona end-to-end com serviços reais

## 6. Testes Corrigidos Nesta Retrospectiva

### Fix 1: test_global_timeout (test_bulas_med.py)
**Problema:** Redis cache retornava resultado real para "Amoxicilina" (cacheado por script de benchmark), bypass completo do timeout wrapper.
**Fix:** `@patch("workflows.whatsapp.tools.bulas_med.cache_service")` com `mock_cache.get = AsyncMock(return_value=None)`.

### Fix 2: test_config_service_returns_error_fallback (test_error_fallback.py)
**Problema:** `ConfigService.get()` agora usa Redis cache-aside (adicionado no Sprint 8). Teste tentava conectar ao Redis real.
**Fix:** `@patch("workflows.services.config_service.get_redis_client")` com mock retornando cache miss.
