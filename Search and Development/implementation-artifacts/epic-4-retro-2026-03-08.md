## 3. O Que Pode Melhorar

### 3.1 Over-Mocking Persiste (Risco Sistêmico)
**Severidade: CRÍTICA**

A race condition H1 do token bucket escapou dos testes unitários porque mocks isolam completamente a lógica de concorrência do Redis. Testes de integração com Redis real teriam capturado automaticamente. Mesmo problema identificado no Epic 1 (cache key mismatch) — padrão recorrente.

**Diretriz do Rodrigo:** "Mockar o necessário, testar o real quando possível."
